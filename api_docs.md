# Документация API

## Пользователи и Аутентификация

### /api/login POST (По сути валидация)
Выполняет вход пользователя в систему.

**Запрос (тело, `application/x-www-form-urlencoded`):**
- `Email: str` - Электронная почта пользователя.
- `PasswordHash: str` - Хеш пароля пользователя.

**Ответ:**
- `200 OK`: `true` - В если аккаунт существует и пароль верный.
- `403 Forbidden`: `false` - Если пользователь с такими данными не найден или пароль неверный.

### /api/register POST (Регистрация нового пользователя)
Создает нового пользователя в системе.

**Запрос (тело, `application/x-www-form-urlencoded`):**
- `Email: str` - Электронная почта (должна быть уникальной).
- `PasswordHash: str` - Хеш пароля.
- `Firstname: str` - Имя.
- `Surname: str` - Фамилия.
- `Lastname: str` (опционально) - Отчество.
- `Description: str` (опционально) - Описание пользователя.
- `Location: str` (опционально) - Местоположение.

**Ответ:**
- `200 OK`: `true` - В случае успешной регистрации.
- `409 Conflict`: `false` - Если регистрация не удалась (например, пользователь с таким `Email` уже существует).

### /api/user/{user_id} GET (Получение профиля пользователя)
Возвращает публичную информацию о пользователе (имя, описание, местоположение).

**Запрос (path-параметр):**
- `user_id: int` - Уникальный идентификатор пользователя.

**Ответ:**
- `200 OK`: `{"id": 1, "FirstName": "Test", "Description": "...", ...}` - Объект с данными пользователя.
- `404 Not Found`: Если пользователь с таким ID не найден.

## Животные

### /api/animal GET (Поиск животных)
Возвращает список животных, отфильтрованный по заданным параметрам.

**Запрос (query-параметры):**
- `Type: str` (опционально) - Тип животного (например, "Cat", "Dog").
- `Breed: str` (опционально) - Порода.
- `Age: int` (опционально) - Ориентировочный возраст.
- `Sterealized: bool` (опционально) - Статус стерилизации (`true` или `false`).
- `limit: int` (опционально, по умолчанию 20) - Количество записей для возврата (макс. 100).
- `offset: int` (опционально, по умолчанию 0) - Количество записей для пропуска (для пагинации).

**Ответ:**
- `200 OK`: `[{"id": 1, "Type": "Cat", ...}, ...]` - Массив объектов, представляющих найденных животных. Если ничего не найдено, вернется пустой массив `[]`.

### /api/animal POST (Создание животного с изображением)
Создает новую запись о животном и загружает его изображение на сервер.

**Запрос (тело, `multipart/form-data`):**
- `Type: str` - Тип животного.
- `Breed: str` - Порода.
- `Name: str` - Имя (кличка).
- `Description: str` - Описание.
- `OrientatedAge: int` - Ориентировочный возраст.
- `Cost: int` - Стоимость (или 0, если бесплатно).
- `Sterealized: bool` - Статус стерилизации (`true` или `false`).
- `image: File` - Файл изображения.
- `Email: str` - Электронная почта пользователя (для валидации).
- `PasswordHash: str` - Хеш пароля пользователя (для валидации).

**Ответ:**
- `200 OK`: `{"message": "Animal created successfully", "image_path": "/uploads/filename.jpg"}` - В случае успешного создания.
- `403 Forbidden`: `{"error": "Unauthorized"}` - В случае неверных данных авторизации.
- `409 Conflict`: `{"error": "Animal with this name already exists"}` - Если животное с таким именем уже существует.
- `422 Unprocessable Entity`: Если какие-то поля не были предоставлены.

### /api/user/{user_id}/animals GET (Поиск животных пользователя)
Возвращает список животных, которые принадлежат конкретному пользователю.

**Запрос (query-параметры):**
- `limit: int` (опционально, по умолчанию 20) - Количество записей.
- `offset: int` (опционально, по умолчанию 0) - Смещение.

**Ответ:**
- `200 OK`: `[{"id": 1, "Type": "Cat", ...}, ...]` - Массив объектов животных.

### /api/animal/{animal_id} GET (Получение одного животного по ID)
Возвращает детальную информацию о конкретном животном.

**Запрос (path-параметр):**
- `animal_id: int` - Уникальный идентификатор животного.

**Ответ:**
- `200 OK`: `{"id": 1, "Type": "Cat", ...}` - Объект с данными животного.
- `404 Not Found`: Если животное с таким ID не найдено.

### /api/animal/{animal_id} DELETE (Удаление животного)
Удаляет животное из базы данных. Требует авторизации владельца.

**Запрос (тело, `application/x-www-form-urlencoded`):**
- `Email: str` - Email владельца для авторизации.
- `PasswordHash: str` - Хеш пароля владельца для авторизации.

**Ответ:**
- `204 No Content`: В случае успешного удаления.
- `403 Forbidden`: Если пользователь не является владельцем животного.
- `404 Not Found`: Если животное с таким ID не найдено.

## Сборы средств и пожертвования

### /api/fundraisers GET (Получение списка сборов)
Возвращает список всех активных сборов средств.

**Запрос (query-параметры):**
- `limit: int` (опционально, по умолчанию 20) - Количество записей.
- `offset: int` (опционально, по умолчанию 0) - Смещение.

**Ответ:**
- `200 OK`: `[{"id": 1, "CreatorUserID": 1, ...}, ...]` - Массив объектов сборов.

### /api/fundraiser POST (Создание сбора средств)
Создает новый сбор средств, который может быть привязан к животному.

**Запрос (тело, `multipart/form-data`):**
- `TargetAmount: int` - Целевая сумма сбора.
- `Description: str` - Описание/цель сбора.
- `AnimalID: int` (опционально) - ID животного, для которого открыт сбор.
- `Email: str` - Email создателя сбора для авторизации.
- `PasswordHash: str` - Хеш пароля создателя.

**Ответ:**
- `200 OK`: `{"id": 1, ...}` - Объект созданного сбора.
- `403 Forbidden`: В случае неверных данных авторизации.

### /api/fundraiser/{fundraiser_id}/donate POST (Сделать пожертвование)
Позволяет авторизованному пользователю сделать пожертвование в рамках активного сбора.

**Запрос (тело, `multipart/form-data`):**
- `Amount: int` - Сумма пожертвования.
- `Email: str` - Email пользователя, совершающего пожертвование.
- `PasswordHash: str` - Хеш пароля.

**Ответ:**
- `200 OK`: `{"message": "Donation successful"}` - В случае успеха.
- `400 Bad Request`: Если сбор не найден, неактивен или произошла ошибка.
- `403 Forbidden`: В случае неверных данных авторизации.

### /api/user/{user_id}/donations GET (Получение истории пожертвований)
Возвращает историю пожертвований конкретного пользователя.

**Запрос (query-параметры):**
- `limit: int` (опционально, по умолчанию 20) - Количество записей.
- `offset: int` (опционально, по умолчанию 0) - Смещение.
